# Project Structure

This project follows a feature-based, layered architecture to ensure a clear separation of concerns, making the codebase scalable and easy to maintain.

```
.
├── docker/
│   ├── api/
│   │   └── Dockerfile          # Dedicated Dockerfile for the API service
│   └── db/
│       └── Dockerfile          # Dedicated Dockerfile for the Database service
├── prisma/
│   ├── migrations/             # Auto-generated by 'prisma migrate'
│   ├── schema.prisma           # The single source of truth for the database schema
│   └── seed.ts                 # Script for seeding the database with initial data
├── requirements/               # Project requirements or related documents
├── specs/                      # Original project specifications
├── src/
│   ├── api/
│   │   ├── auth/               # Handles authentication (login)
│   │   ├── interview/          # Handles all logic for Interviews
│   │   ├── interview-comment/  # Handles logic for Comments
│   │   ├── interview-history/  # Handles logic for viewing History
│   │   └── user/               # Handles user management (Admin)
│   ├── config/
│   ├── core/
│   ├── tests/
│   ├── app.ts                  # Express application setup and middleware registration
│   └── server.ts               # Server initialization and startup
├── .dockerignore
├── .env.example                # Example environment file
├── .gitignore
├── docker-compose.yml          # Main Docker Compose file for orchestration
├── eslint.config.js            # Configuration for the ESLint code linter
├── jest.config.js              # Jest test runner configuration
├── package.json
├── package-lock.json
├── README.md                   # Main project documentation
└── tsconfig.json               # TypeScript compiler configuration
```
